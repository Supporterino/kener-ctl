## Why

The `orhun/git-cliff-action@v4` wrapper causes spurious CI failures when writing changelog output to `$GITHUB_OUTPUT` — the formatted content gets interpreted as workflow commands by the Actions runner, producing `Unable to process file command 'output'` errors. GitHub's built-in release notes generation is simpler, zero-config, and immune to this class of bugs.

## What Changes

- Remove the `orhun/git-cliff-action@v4` step from the release workflow
- Remove `cliff.toml` changelog configuration file from the repository root
- Remove `CHANGELOG.md` (generated artifact, no longer produced)
- Enable `generate_release_notes: true` on `softprops/action-gh-release@v2`
- Remove `body_path: CHANGELOG.md` from the release step

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `standalone-releases`: The "Automated changelog generation" requirement changes from git-cliff with `cliff.toml` to GitHub's auto-generated release notes. The cliff.toml scenario and related git-cliff scenarios are replaced with a requirement that release notes are auto-generated by GitHub.

## Impact

| File | Change |
|---|---|
| `.github/workflows/release.yml` | Replace git-cliff step with `generate_release_notes: true` |
| `cliff.toml` | Delete |
| `CHANGELOG.md` | Delete |
| `openspec/specs/standalone-releases/spec.md` | Update changelog requirement |

No code, no dependencies, no breaking changes to the CLI or its behavior.
